# Contributing to Vibeflow

Thanks for your interest in contributing to Vibeflow! This document explains how to get involved.

## How to contribute

### Reporting bugs

Open an issue using the **Bug Report** template. Include:
- Edition affected (Copilot, Cursor, Claude Code, or CLI)
- Steps to reproduce
- Expected vs actual behavior

### Suggesting features

Open an issue using the **Feature Request** template. Describe the problem you're solving, not just the solution you want.

### Submitting code

1. **Fork** the repo and create a branch from `main`
2. Make your changes
3. Test your changes manually (see below)
4. Submit a **Pull Request** using the PR template

### What we accept

- Bug fixes (always welcome)
- Documentation improvements (always welcome)
- Prompt/methodology improvements (discuss in an issue first)
- New edition support (discuss in an issue first)
- CLI improvements (discuss in an issue first)

## Project structure

```
vibeflow/
├── copilot/        # GitHub Copilot edition (prompts, agents, skills)
├── cursor/         # Cursor edition (rules, skills)
├── claude-code/    # Claude Code edition (commands, agents, skills)
├── cli/            # npm installer (setup-vibeflow)
├── site/           # Astro landing page (i18n: en, pt-br)
├── MANUAL.md       # Full methodology docs (PT-BR)
├── CHANGELOG.md    # Version history
└── README.md       # Project overview (EN)
```

### Key concepts

- **3 editions, 1 methodology** — Copilot, Cursor, and Claude Code share the same pipeline. Changes to methodology should be applied to all 3 editions.
- **Source of truth** — Copilot prompts (`copilot/github/prompts/`) are the reference. Cursor skills and Claude Code commands mirror them.
- **`.vibeflow/`** — The project knowledge base generated by `analyze`. Not committed by default.

## Development

### Site (Astro)

```bash
cd site
npm install
npm run dev     # http://localhost:4321
npm run build   # production build
```

### CLI

```bash
cd cli
node index.js --copilot   # test copilot install
node index.js --cursor    # test cursor install
```

### Prompts / Commands / Skills

These are Markdown files. Edit directly and test in the respective agent:
- **Copilot**: `copilot/github/prompts/*.prompt.md`
- **Cursor**: `cursor/skills/*/SKILL.md`
- **Claude Code**: `claude-code/commands/*.md`

## Code style

- **Commits**: Use [Conventional Commits](https://www.conventionalcommits.org/) — `feat:`, `fix:`, `docs:`, `chore:`
- **Language**: Code and commits in English. Documentation can be bilingual (EN + PT-BR).
- **Prompts**: Keep prompts concise. Lead with rules, not explanations. Use code examples.

## Sync workflow

Changes to `claude-code/` are auto-synced to the [vibeflow-claude](https://github.com/pe-menezes/vibeflow-claude) distribution repo via GitHub Actions. You don't need to update that repo manually.
